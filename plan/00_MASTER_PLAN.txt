  Stílus: 2D top-down pixel art (32×32 tile, 32×48 karakter)

    Diablo II → combat, loot rendszer, affix-ek, endgame grind
    World of Warcraft → class design, progression, raid mechanika
    Path of Exile → gem rendszer, skill tree mélység

  1.1 Egymondatos pitch
  ──────────────────
    "Egy sötét fantasy pixel art ARPG ahol 1-4 játékos fedez fel egy
    procedurálisan generált világot, fejleszti karakterét 3 osztály
    egyikében, és raid bossokat győz le – teljesen AI-generált

  3.1 Implementációs sorrend (mi mire épül)
  ──────────────────

  Layer 0 – ALAP (előfeltétel nélkül):
    [10] Godot Architecture → projekt struktúra, autoload-ok, event bus
    [AI] LoRA training → stílusos sprite generálás alapja

  Layer 1 – CORE (Layer 0 kell hozzá):
    [01] Skill Tree → igényli: Godot Arch (player entity, combat system)
    [02] Procedural World → igényli: Godot Arch (chunk system, tilemap)
    [06] Enemy AI → igényli: Godot Arch (entity base, pathfinding)

  Layer 2 – GAMEPLAY (Layer 0 + 1 kell hozzá):
    [08] Loot System → igényli: Skill Tree (stat rendszer), Enemy AI (drop)
    [07] Dungeon Gen → igényli: Proc World (biome), Enemy AI (spawn)
    [03] Boss Design → igényli: Enemy AI (BT), Skill Tree (damage calc)

  Layer 3 – DEPTH (Layer 0-2 kell hozzá):
    [09] Gem System → igényli: Loot System (item socket), Economy (gold)
    [04] Economy → igényli: Loot (item value), Boss (rare drops)

  Layer 4 – MULTIPLAYER (minden másra épül):
    [05] Networking → igényli: MINDEN korábbi rendszer (sync-elni kell)

    [11] Roadmap → ütemezi az összes fenti rendszer implementálását

  V. TELJES TECHNIKAI SPECIFIKÁCIÓ
================================================================================

  5.1 Technológiai stack
  ──────────────────

    ┌──────────────────┬──────────────────────────────────────────────┐
    │ Technológia      │ Részletek                                    │
    ├──────────────────┼──────────────────────────────────────────────┤
    │ Motor            │ Godot 4.x (stable)                           │
    │ Nyelv            │ GDScript (100%)                              │
    │ Renderer         │ Compatibility (OpenGL, 2D optimalizált)      │
    │ Multiplayer API  │ ENetMultiplayerPeer (high-level)             │
    │ Noise            │ FastNoiseLite (beépített)                    │
    │ Pathfinding      │ NavigationServer2D (beépített)               │
    │ AI generálás     │ Stable Diffusion XL + ComfyUI                │
    │ LoRA training    │ Kohya_ss                                     │
    │ Post-process     │ Aseprite (pixel art cleanup)                 │
    │ Verziókezelés    │ Git + GitHub/GitLab                          │
    │ Hang             │ .ogg (zene), .wav (SFX)                      │
    │ Mentés formátum  │ JSON (enkryptált opcionálisan)               │
    └──────────────────┴──────────────────────────────────────────────┘

  5.2 Viewport és renderelés
  ──────────────────

    Viewport méret: 640×480 logikai pixel
    Megjelenítés: 1280×960 (2× upscale) vagy magasabb + integer scaling
    Tile méret: 32×32 pixel
    Karakter sprite: 32×48 pixel (32 széles, 48 magas)
    Enemy sprite: 32×48 (alap), 48×64 (elite), 64-192×64-192 (boss)
    Látható terület: ~40×30 tile (20×15 chunk)
    FPS cél: 60 FPS (minimum 30 FPS)

  5.3 Multiplayer specifikáció
  ──────────────────

    Modell: Host-authoritative (peer-to-peer)
    Max játékos: 4 (1 host + 3 client)
    Tick rate: 20/sec (50ms update interval)
    Sávszélesség: ~5-10 KB/sec (host upstream)
    Port: 7777 (TCP/UDP via ENet)
    Reconnect timeout: 5 perc
    Latency cél: <150ms elfogadható
    Nincs: dedikált szerver, host migration

  5.4 Save/Load rendszer
  ──────────────────

    Formátum: JSON
    Slot-ok: 3 manuális + 1 autosave
    Autosave: biome váltásnál, dungeon ki/belépésnél
    Mentett adat: pozíció, inventory, skill-ek, quest állapot,
                  world seed, felfedezett terület, economy, stash

  5.5 Collision rétegek
  ──────────────────

    Layer 1: Solid (fal, akadály)
    Layer 2: Player
    Layer 3: Enemy
    Layer 4: NPC
    Layer 5: Player Hitbox
    Layer 6: Enemy Hitbox
    Layer 7: Player Hurtbox
    Layer 8: Enemy Hurtbox
    Layer 9: Projectile
    Layer 10: Pickup (loot, resource)

  5.6 Autoload Singleton-ök
  ──────────────────

    GameManager     → játék állapot (menü/loading/playing/paused)
    EventBus        → globális signal-ok (~25 signal)
    SaveManager     → save/load logika
    AudioManager    → zene/SFX lejátszás, crossfade
    SceneManager    → scene váltás transition-ökkel
    NetworkManager  → multiplayer kapcsolat
    Constants       → globális konstansok, enum-ok
    InputManager    → input mapping + state

================================================================================
  VII. AI ASSET PIPELINE ÖSSZEFOGLALÓ
================================================================================

  7.1 Hardware követelmények
  ──────────────────

    GPU: NVIDIA RTX 3060+ (min 8GB VRAM, ajánlott 12GB+)
    SDXL model: ~6.5GB VRAM használat
    LoRA training: ~10-12GB VRAM (Kohya_ss)
    Generálási sebesség: ~5-15 sec/image (512×512, 20 step)

  7.2 Pipeline lépések
  ──────────────────

    1. STÍLUS DEFINÍCIÓ
       └─ 20-30 referencia pixel art gyűjtés
       └─ "ashenfall_style" LoRA training (Kohya_ss)
       └─ LoRA paraméterek: dim 32, alpha 16, 1500-2000 step

    2. GENERÁLÁS (ComfyUI batch workflow)
       └─ Prompt template per asset típus
       └─ Negative prompt: 3D, photorealistic, blurry, text
       └─ Batch: 20-50 kép / session → válogatás (top 20-30%)
       └─ Seed tracking: jó eredmények seed-jei mentve

    3. POST-PROCESS (Aseprite)
       └─ Méretezés: pontos pixel méret (32×32, 32×48, stb.)
       └─ Palette lock: biome / asset specifikus paletta
       └─ Frame cut: sprite sheet → egyedi frame-ek
       └─ Cleanup: pixel hibák javítás, outline consistency
       └─ Animáció: frame sorrend, timing

    4. GODOT IMPORT
       └─ SpriteFrames resource létrehozás
       └─ Animáció beállítás (FPS, loop)
       └─ Teszt: in-game megjelenés ellenőrzés

  7.3 Prompt template példák
  ──────────────────

    Karakter: "pixel art character sprite sheet, [class], [action],
              32x48, dark fantasy, top-down view, ashenfall_style LoRA"

    Tileset: "pixel art tileset, [biome], 32x32 grid, seamless edges,
             dark fantasy atmosphere, ashenfall_style LoRA"

    Item: "pixel art item icon, [item_name], 32x32, dark fantasy,
           clean background, detailed, ashenfall_style LoRA"

    Boss: "pixel art boss sprite, [name], [size]px, dark fantasy,
           intimidating, detailed, top-down perspective, ashenfall_style"

    T2 Dungeon Boss → 3K-8K HP, 2-3 Phase, 1-2 player, Rare-Epic loot
    T3 World Boss   → 15K+ HP, 3-4 Phase, 2-4 player, Epic-Leg. loot
    T4 Raid Boss    → 50K+ HP, 4-5 Phase, 4 player, Legendary loot

    6 tier: Chipped → Flawed → Normal → Flawless → Perfect → Radiant
    Upgrade: 3× azonos → 1× magasabb tier
    Socket: 0-4 / item (rarity-függő)

    Gold: fő valuta (enemy kill, quest, trade)
    Dark Essence: prémium (boss kill, weekly challenge, achievement)
    Relic Fragment: endgame (T3-T4 boss, raid completion)

    Host-Client (ENet), Port 7777, Max 4 player, 20 tick/sec
    Személyes loot, Boss HP scaling (+50%/+100%/+150% per player)
    Reconnect: 5 perc timeout, nincs host migration
