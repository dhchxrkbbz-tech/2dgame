  4.1 Szoftver Stack (v2 – csak Godot-natív)
  ────────────────────────────────────────────

  4.2 Hardware Követelmény (v2)
  ──────────────────────────────
    CPU: bármilyen modern processzor
    RAM: 8GB+ (Godot fut, Claude API hívások)
    Storage: 10GB (PNG exportok + projekt)
    GPU: NEM szükséges (nincs AI generálás)
    Internet: Claude API hívásokhoz

  4.3 Generáló Engine: Godot Image API
  ──────────────────────────────────────
    Legfontosabb metódusok:
      Image.create(width, height, mipmaps, format)
      image.set_pixel(x, y, color)
      image.get_pixel(x, y)
      image.fill(color)
      image.save_png(path)
      ImageTexture.create_from_image(image)
      SpriteFrames.new() / add_animation() / add_frame()

    PixelArtBase helper osztály metódusai:
      fill_rect(img, x, y, w, h, color)
      draw_outline(img, color, thickness)
      draw_circle(img, cx, cy, r, color)
      draw_line_px(img, x0, y0, x1, y1, color)
      apply_palette(img, palette_dict) → Image
      shift_image(img, dx, dy) → Image
      flip_horizontal(img) → Image
      add_noise(img, rng, strength, seed) → void

  4.6 Post-Processing (v2 – minimális, Godot-natív)
  ───────────────────────────────────────────────────
    1. GDScript generate → Image objektum
    2. draw_outline() hívás (minden sprite-on kötelező)
    3. image.save_png(path) exportálás
    4. Godot import: nearest filter beállítás
    5. TileSet terrain rules manuális setup (vagy GDScript tool)
    6. AnimatedSprite2D: SpriteFrames resource hozzárendelés
    7. Collision shape: manual adjust (ha GDScript tool nem csinálja)

  4.7 Három generálási módszer (v2)
  ──────────────────────────────────
    Method A: Teljesen procedurális (egyszerű sprite-ok)
      → 64×64-es vagy kisebb tile-ok, ikonok
      → Minden pixel determinisztikus (seed-alapú random)
      → Egyik leggyorsabb módszer, ~50 sprite/session

    Method B: Moduláris összeépítés (karakter sprite-ok)
      → Test részek külön draw függvényekben
      → pl. _draw_head(), _draw_torso(), _draw_weapon()
      → Animáció: részek offset-elve frame_idx alapján

    Method C: Keretrajzolás + részletek (boss-ok)
      → Először a forma (nagy fill_rect-ek)
      → Utána részletek rétegesen (textúra, outline, glow)
      → Phase változások: külön palette + overlay layer

  4.8 Godot Import Pipeline (v2)
  ───────────────────────────────
    1. GDScript generator .gd fájl → generators/ mappa
    2. batch_export_tool.gd futtatása
    3. PNG-ek → assets/ alá exportálva
    4. Godot auto-detects new .png fájlokat
    5. Import settings: nearest, no compression, RGBA8
    6. SpriteFrames .tres resource: GDScript-ből vagy kézzel Godot editorban
    7. AnimatedSprite2D: SpriteFrames hozzárendelve
